## 实验 1：ALNS 多随机种子鲁棒性实验

### 背景
当前论文仅使用固定随机种子 seed=42 运行 ALNS 一次。作为随机搜索算法，必须验证结果对随机种子的敏感性。

### 实验设计

**随机种子集合**：使用 10 个种子 `{1, 7, 13, 23, 42, 56, 71, 88, 97, 123}`，其中 seed=42 为论文原始结果，可直接复用。

**运行实例**：全部 48 个实例，每个实例运行 10 次（10 个种子）。

**运行时间**：每次运行 60 秒（与论文一致）。

**记录指标**（每个实例 × 每个种子）：
- `Obj`：ALNS 返回的目标函数值
- `BKS`：最佳已知解（沿用论文表 5.2 中四种方法的最优可行解）

### 输出表格

**表 A：ALNS 多种子汇总结果**（替换/补充论文表 5.2 中 ALNS 列）

| 实例 | BKS | Obj_mean | Obj_best | Obj_worst | Std | Dev_mean(%) | Dev_best(%) | Dev_worst(%) |
|------|-----|----------|----------|-----------|-----|-------------|-------------|--------------|
| 1_10_3_2 | ... | ... | ... | ... | ... | ... | ... | ... |
| ... | | | | | | | | |

其中：
- `Obj_mean` = 10 次运行目标值的均值
- `Obj_best` = 10 次运行中的最优目标值  
- `Obj_worst` = 10 次运行中的最差目标值
- `Std` = 10 次运行目标值的标准差
- `Dev_mean(%)` = (Obj_mean - BKS) / BKS × 100
- `Dev_best(%)` = (Obj_best - BKS) / BKS × 100
- `Dev_worst(%)` = (Obj_worst - BKS) / BKS × 100

**表 B：ALNS 鲁棒性汇总统计**

| 指标 | 值 |
|------|-----|
| 平均 Dev_mean(%) | |
| 平均 Dev_best(%) | |
| 平均 Dev_worst(%) | |
| 平均 Std | |
| 平均变异系数 CV(%) = Std/Obj_mean × 100 | |
| Obj_best 达到 BKS 的实例数（10 次中至少 1 次） | |
| 所有 10 次均达到 BKS 的实例数 | |

### 代码修改要点

在 ALNS 主类中：
1. 将随机种子从硬编码 `42` 改为从命令行参数或循环变量读取
2. 外层循环遍历 10 个种子，内层遍历 48 个实例
3. 每次运行记录目标值到 CSV 文件
4. 运行完成后用脚本汇总统计

```java
// 伪代码示意
int[] seeds = {1, 7, 13, 23, 42, 56, 71, 88, 97, 123};
for (int seed : seeds) {
    Random rng = new Random(seed);
    // ... run ALNS with rng, time limit 60s ...
    // record: instance_name, seed, objective_value
}
```

### 结果用途
- 表 A 替换论文 5.2.3 节中 ALNS 的单种子结果
- 表 B 作为新增表格放入 5.2.3 节
- 更新 5.1.3 节末尾关于种子的说明文字（见文本替换文档）

## 实验 2：ALNS 运行时间敏感性实验

### 背景
论文中 ALNS 固定运行 60 秒。审稿人可能关心：增加时间预算后 ALNS 解质量能提升多少？

### 实验设计

**运行时间集合**：`{30, 60, 120, 300}` 秒

**随机种子**：使用实验 1 中表现最稳定的种子（例如 seed=42），或取 10 个种子的中位数结果。建议直接使用 seed=42 以保持与消融实验等的一致性。

**运行实例**：选取有代表性的 16 个实例——LBBD 已证明最优的 32 个实例中选 8 个 + n=15 的 8 个较大实例。具体为：
- n=10 选 8 个：`1_10_6_3, 3_10_6_3, 1_10_9_2, 3_10_9_2, 1_10_9_3, 3_10_9_3, 1_10_3_2, 3_10_3_2`
- n=15 选 8 个：`1_15_6_2, 3_15_6_2, 1_15_6_3, 3_15_6_3, 1_15_9_2, 3_15_9_2, 1_15_9_3, 3_15_9_3`

### 输出表格

**表 C：ALNS 运行时间敏感性**

| 实例 | BKS | Dev@30s(%) | Dev@60s(%) | Dev@120s(%) | Dev@300s(%) |
|------|-----|------------|------------|-------------|-------------|
| 1_10_6_3 | ... | ... | ... | ... | ... |
| ... | | | | | |
| **平均** | | | | | |

### 代码修改要点
将 ALNS 的时间限制参数化，循环运行 4 个时间设定。

### 结果用途
- 作为新增表格放入 5.2.3 节末尾
- 用于讨论 ALNS 的"时间-质量"权衡特性
